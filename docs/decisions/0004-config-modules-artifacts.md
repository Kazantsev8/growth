# 0004 — Модель config.md: модули и артефакты

- **Статус:** принято
- **Дата:** 2026-06-07 (ретроспективно)

## Контекст
Нужен расширяемый реестр направлений и их содержимого, читаемый приложением и правимый вручную.

## Решение
`config.md` — YAML-frontmatter со списком `modules`. Поля модуля: `id, title, icon, path, order, enabled`.
Далее одно из двух:
- **один артефакт:** у модуля `kind` (+ опц. `file`); путь = `path` (или `path/file`);
- **несколько артефактов:** `artifacts: [...]`, каждый `{id, title, kind, file}`; путь = `<path>/<file>`.
Артефакт без `file`/`path` → graceful-заглушка. Конфиг правится вручную, приложение только читает.

## Последствия
- (+) Добавить направление/артефакт — правка одного YAML.
- (+) Один `kind` может рендерить несколько внутренних вкладок (см. Sport, ADR 0006).
- Снимок структуры — `docs/config.reference.md`; живой конфиг — на Mail.ru.
