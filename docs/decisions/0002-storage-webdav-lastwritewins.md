# 0002 — Хранилище: Markdown на WebDAV, last-write-wins

- **Статус:** принято
- **Дата:** 2026-06-07 (зафиксировано ретроспективно)

## Контекст
Нужно бесплатное хранилище без бэкенда, совместимое с Obsidian-vault пользователя. Один пользователь,
несколько устройств; Obsidian синкает vault редко/вручную.

## Решение
Источник истины — Markdown-файлы на Mail.ru WebDAV. Стратегия конфликтов —
**last-write-wins по `getlastmodified`**; `etag` Mail.ru НЕ доверять (туда кладутся первые байты содержимого).
Правка артефакта = перезапись файла целиком. Защиту от одновременной записи не строим.

## Последствия
- (+) Никакого бэкенда; данные читаемы и редактируемы в Obsidian.
- (+) Простая модель записи.
- (−) Возможна потеря при гонке (приемлемо для одного пользователя).
- Следствие для парсеров: нужно толерантно сохранять неизвестное содержимое (см. ADR 0008).
