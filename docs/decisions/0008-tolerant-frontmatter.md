# 0008 — Толерантный парсинг (сохраняем неизвестное)

- **Статус:** принято
- **Дата:** 2026-06-08

## Контекст
Файлы редактируются и приложением, и в Obsidian. Если парсер знает только свой формат, ручные правки
(новые ключи frontmatter, свободный текст) теряются при перезаписи (см. ADR 0002 — запись файла целиком).

## Решение
Парсеры толерантны: читают известное, **сохраняют неизвестное** и возвращают при записи.
Для frontmatter — через `jsyaml.load`/`jsyaml.dump` всего объекта (неизвестные ключи едут с ним).
Для тела — сохраняем intro-текст до структуры и «хвост» после (best-effort).

## Последствия
- (+) Правки из Obsidian выживают; формат расширяем без миграций (напр. будущие `priority/due/tags`).
- (−) Глубокая произвольная разметка внутри структур может не сохраниться побайтно (документировано в specs).
