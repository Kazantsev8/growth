# 0006 — Sport: единый kind с двумя вкладками

- **Статус:** принято
- **Дата:** 2026-06-13

## Контекст
Sport — это трекер тренировок (журнал + графики) И список редактируемых программ. Перечень видов спорта
общий для обоих и используется в каскаде (переименование/удаление вида). Дробить на два независимых
артефакта = разделять общее состояние между рендерерами.

## Решение
Один `kind: sport`, внутри две вкладки (Трекер / Программы), общий перечень видов и общий код.
Журнал — `Sport_log.md` (frontmatter `types:` + таблица `Date|Type|Program|Note`). Программы —
редактируемые `.md` в `Workouts/`, у каждой `type` во frontmatter. Каскад по видам проходит по журналу
и файлам программ.

## Последствия
- (+) Простой общий стейт, цельный каскад.
- (−) Один рендерер крупнее; компенсируется ленивой загрузкой (ADR 0001).
- Детали — [specs/sport.md](../specs/sport.md) и [guides/Sport.md](../guides/Sport.md).
