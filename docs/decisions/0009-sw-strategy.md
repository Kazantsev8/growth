# 0009 — Стратегия service worker при многофайловости

- **Статус:** принято (реализация — Фаза 3)
- **Дата:** 2026-06-13

## Контекст
После дробления на много `.js`/`.css` офлайн должен сохраниться, а свежий код — приезжать при онлайне
(пользователь часто пушит). Контент-хэшей имён файлов нет (без сборки).

## Решение
- install: precache shell (`index.html`, все `css/*`, ядро `js/*`, вендоры, шрифты, иконки).
- fetch: навигации/HTML → network-first; same-origin `*.js`/`*.css` (вкл. ленивые kind) → network-first +
  запись в рантайм-кэш; шрифты/вендоры/иконки → cache-first; не-GET → мимо SW.
- Версия кэша `growth-vN`; **bump при изменении набора файлов**. Корректность не зависит от bump
  (онлайн всегда network-first); bump лишь чистит старое.

## Последствия
- (+) Свежий код при онлайне без обязательного bump каждую итерацию; офлайн работает после первого визита.
- (−) Небольшая сетевая задержка на первый показ kind при онлайне (приемлемо).
- Альтернатива на будущее: stale-while-revalidate для `*.js`/`*.css` (мгновенно + фон-обновление) — отдельным ADR при необходимости.
