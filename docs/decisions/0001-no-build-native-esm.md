# 0001 — Без сборки, нативные ES-модули

- **Статус:** принято
- **Дата:** 2026-06-13

## Контекст
Код вырос в монолит `app/index.html` (~1873 строки). Нужна многомодульная архитектура,
но базовый принцип проекта — **без сборки**: CF Pages раздаёт `app/` как статику, деплой = `git push`,
проверка = `node --check`. Вводить бандлер (Vite/esbuild) — значит package.json, node_modules,
build-конфиг CF Pages и смена workflow.

## Решение
Разбить код на нативные ES-модули (`<script type="module">` + относительные `import`), **без бандлера**.
Тяжёлые kind-модули грузить лениво через `import()`. Вендоры (`js-yaml`, `marked`) оставить
классическими `<script>`-глобалами, доступ централизовать в `core/vendor.js`.

## Последствия
- (+) Сохраняется «push → деплой», `node --check`, отладка по реальным файлам.
- (+) Чистые слои и границы модулей; легко добавлять kind.
- (−) Нет минификации/tree-shaking (для личного приложения некритично).
- (−) Нужно вести список файлов в `sw.js`; ESM не работает по `file://` (тест только по http).

## Альтернативы
Бандлер (Vite) — отвергнут: ломает «без сборки» и усложняет хендоф/деплой ради малой выгоды на этом масштабе.
