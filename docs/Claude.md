## ⚠️ ТРИГГЕР: обязательное обновление документации

**При любом изменении состояния проекта ты ОБЯЗАН обновить документацию, прежде чем закончить работу.** Это не опционально. Конкретно:

- добавлено/удалено направление или артефакт → обнови `config.md`, этот `Claude.md` и соответствующий гайд в `guides/`;
- изменился формат хранения, схема `config.md` или появилась новая конвенция → обнови раздел «Архитектура»/«Конвенции» здесь и затронутые гайды;
- добавлен новый `kind` артефакта → опиши его в разделе «Артефакты (kind)» и в гайде направления, где он используется;
- изменилась инфраструктура (воркер, хостинг, домен) → обнови раздел «Инфраструктура».

Эти документы (`Claude.md` + `guides/*`) — **источник истины для AI-хендофа**. Следующая сессия доверяет им, а не догадкам. Устаревший гайд хуже отсутствующего.

## Что это

Личный трекер саморазвития одного пользователя. Направлений может быть N (English, iOS, Android, Sport, …). Каждое направление — контейнер «артефактов» (словарь, справочник, чеклист, backlog заметок и т.п.). Сверхцель — вернуть «продуктивное трение» в обучении и не деградировать; инструмент строит Claude, пользователь им пользуется.

## Инфраструктура (всё бесплатно, без бэкенда)

- **Клиент:** одностраничный PWA на ванильном JS (один файл `index.html`). Хостинг — **Cloudflare Pages** (`https://growth.pages.dev`), ставится на iPhone как PWA.
- **Хранилище (источник истины):** Markdown-файлы на **Mail.ru WebDAV** (`https://webdav.cloud.mail.ru`) — это облако пользователя.
- **Мост:** **Cloudflare Worker** `growth-dav` (`https://growth-dav.vans-bleat.workers.dev`) — CORS-прокси перед Mail.ru. Защищён заголовком `X-Proxy-Secret`. Бизнес-логики не содержит.
- **Авторизация:** пароль приложения Mail.ru (Basic auth), вводится в аппе и хранится локально на устройстве. URL воркера, секрет и логин зашиты константами в коде аппа.

## Раскладка файлов

```
/GrowthApp/                  ← служебная папка приложения (Obsidian её не трогает)
  config.md                  ← реестр направлений и артефактов (читается аппом при старте)
  Claude.md                  ← этот файл
  guides/
    English.md               ← гайд по направлению English
/Obsidian Vault/             ← данные, в реальной структуре vault пользователя
  Personal/English/          ← артефакты English (см. guides/English.md)
  Professional/Swift/        ← iOS
  Professional/Android/      ← Android
  Personal/Sport/            ← Sport
```

## Модель config.md

`config.md` — YAML-frontmatter со списком `modules`. Поля модуля: `id, title, icon, path, order, enabled`. Дальше одно из двух:
- **один артефакт:** у модуля есть `kind` (+ при необходимости `file`); данные в `path`;
- **несколько артефактов:** у модуля есть `artifacts: [...]`, каждый артефакт = `{ id, title, kind, file }`, путь к файлу = `<path>/<file>`.

Конфиг в v1 правится **вручную**; приложение его только читает.

## Артефакты (kind) — что реализовано

- **vocabulary** — словарь. Хранится Markdown-таблицей в одном файле; колонки `Word | Translation | Mastered | Example | Note` (`mastered` = `yes`/`no`). В аппе: карточки, поиск, добавление/удаление (запись перезаписывает весь файл).
- **doc** — markdown-справочник. Рендерится как есть (заголовки, таблицы, списки, цитаты). Ссылки оглавления (`#слаг`) работают — заголовкам проставляются id по GitHub-слагу.
- **notes** — backlog заметок. Дат-блоки `## ГГГГ-ММ-ДД ЧЧ:ММ` + текст, новые сверху. В аппе: поле ввода + список с удалением. Используется как канал «идеи → анализ AI → развитие направления».
- **checklist** — список `- [ ]` / `- [x]` + прогресс-бар.
- **weekly-habit** — пока заглушка (показывает файлы), рендерер впереди.

## Синхронизация

Один пользователь. Источник истины — файлы на Mail.ru. Стратегия — **last-write-wins по `getlastmodified`**; `etag` Mail.ru НЕ доверять (туда кладутся первые байты содержимого). Obsidian синкает vault редко/вручную, поэтому защиту от одновременной записи не строим. Правка артефакта = перезапись файла целиком.

## Конвенции

- Пути в WebDAV приложение URL-кодирует само (пробелы → `%20`, кириллица — тоже); в `config.md` пути пишутся по-человечески.
- `GrowthApp/` — единоличная зона приложения; данные направлений живут в `Obsidian Vault/`.
- Имена данных без префиксов `_`/`.` (на случай сторонних синкеров вроде Remotely Save).
