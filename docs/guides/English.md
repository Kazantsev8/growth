# Гайд: направление English

> Документация для AI и для пользователя по работе с направлением English.
> При изменении состава артефактов или форматов — обнови этот файл и `config.md` (см. триггер в `../Claude.md`).

## Назначение

Хаб изучения английского: рабочий словарь + справочники по грамматике + backlog идей. Пользователь возвращается сюда, чтобы повторять и пополнять материал.

## Расположение

Данные: `Obsidian Vault/Personal/English/`
Запись в конфиге: модуль `english` в `GrowthApp/config.md` (с блоком `artifacts`).

## Артефакты

| id | Заголовок | kind | Файл |
|---|---|---|---|
| vocabulary | Vocabulary | vocabulary | `Vocabulary.md` |
| tenses | Tenses | doc | `English_tenses.md` |
| conditionals | Conditionals | doc | `English_conditionals.md` |
| irregular | Irregular Verbs | doc | `English_irregular_verbs.md` |
| backlog | Backlog | notes | `English_backlog.md` |

### vocabulary → `Vocabulary.md`

Markdown-таблица. Колонки строго в порядке:

```
| Word | Translation | Mastered | Example | Note |
|---|---|---|---|---|
| valour | доблесть, отвага | no |  |  |
```

- `Mastered` — `yes`/`no` (также принимаются `x`/`true`/`1`). Управляет бейджем `[MASTERED]`/`[LEARNING]` и счётчиком «N to master».
- `Example` — пример употребления; рисует строку `>` в карточке.
- `Note` — заметка/комментарий.
- В тексте ячеек символ `|` экранируется как `\|`.
- Приложение читает весь файл, рисует карточки (поиск, добавление, удаление), при правке **перезаписывает файл целиком**. Порядок строк в файле сохраняется (сортировка — только на экране: невыученные сверху).
- Масштаб: один файл тянет тысячи строк. Если разрастётся — шардить по первой букве (`a.md`, `b.md`, …), это плановая эволюция формата.

### doc → справочники (tenses / conditionals / irregular)

Обычные markdown-документы: заголовки, таблицы, цитаты, оглавление. Рендерятся как есть. Внутренние ссылки оглавления (`#слаг`) работают — заголовкам проставляются id по GitHub-слагу (lowercase, пробелы→дефис, пунктуация/эмодзи отброшены). Только чтение.

### notes → `English_backlog.md`

Backlog идей по развитию English. Формат — дат-блоки, новые сверху:

```
## 2026-06-07 14:32

текст заметки (может быть многострочным)
```

Приложение: поле ввода + список карточек с удалением, пишет новые сверху.

## Как добавить новый артефакт в English

1. Положи файл в `Obsidian Vault/Personal/English/`.
2. Добавь блок под `english.artifacts` в `config.md`:
   ```
   - id: <slug>
     title: <Заголовок>
     kind: doc            # или vocabulary / notes / checklist
     file: <Имя_файла>.md
   ```
3. Обнови таблицу артефактов в этом гайде (триггер в `../Claude.md`).
   Передеплой аппа не нужен — конфиг читается с сервера; достаточно обновить страницу.

## Цикл развития через backlog

`English_backlog.md` — канал «мысли пользователя → анализ AI → развитие направления». Когда пользователь просит проанализировать backlog: прочитать `English_backlog.md`, сгруппировать идеи, предложить конкретные шаги (новые артефакты, доработки словаря/справочников), реализовать согласованное и обновить этот гайд.
