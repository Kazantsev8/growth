# Sport — трекер тренировок + программы

Направление `sport` (`kind: sport`) — геймификация спорта. Один рендерер с двумя вкладками внутри: **Трекер** и **Программы**. `path` модуля — корень раздела (`Obsidian Vault/Personal/Sport`).

## Раскладка на WebDAV

```
Sport/
  Sport_log.md         ← журнал тренировок + перечень видов спорта
  Workouts/            ← программы тренировок (по файлу на программу)
    Грудь день.md
    Ноги.md
```

Папки/файлы создаются приложением при первом сохранении (`MKCOL`).

## Трекер — `Sport_log.md`

Frontmatter с перечнем видов + Markdown-таблица журнала. Дата — по дню (`YYYY-MM-DD`), несколько тренировок за день допускаются.

```markdown
---
types:
  - Зал
  - Плавание
  - Бег
---

| Date | Type | Program | Note |
|---|---|---|---|
| 2026-06-13 | Зал | [Грудь день](Workouts/Грудь%20день.md) | грудь/трицепс |
| 2026-06-11 | Плавание |  | 1 км |
```

- `types` — общий перечень видов спорта (используется и для отметок, и для программ). Неизвестные ключи frontmatter сохраняются (толерантно).
- `Type` — вид спорта (из перечня).
- `Program` — необязательная ссылка на программу: `[Название](Workouts/<имя>.md)`. При чтении извлекается текст ссылки (имя программы).
- `Note` — заметка.

**Визуализация (SVG/DOM, без внешних либ):**
- **Календарь-хитмап** (26 недель): день делится на вертикальные сегменты по числу тренировок, каждый — цветом своего вида. Палитра яркая, вне сиреневой темы; цвет назначается по индексу вида в `types`. Удалённые виды (в истории) — серым.
- **Столбики по неделям** (12 недель) — число тренировок.
- **Счётчики**: всего / эта неделя / **серия по неделям** (подряд идущие недели с ≥1 тренировкой — учитывает отдых между тренировками).
- **Журнал** — записи по дням (новые сверху), удаление.

## Программы — `Workouts/*.md`

Редактируемые в приложении Markdown-файлы (создание/чтение/правка/удаление). Имя файла = название программы (`<name>.md`). Чтение рендерится через `marked`.

```markdown
---
type: Зал
---

# Грудь / трицепс
- Жим лёжа 4×8
- Разводка гантелей 3×12
```

- `type` — вид спорта программы (необязательно), из общего перечня; показывается бейджем.
- Переименование программы = запись нового файла + удаление старого.

## Виды спорта — целостность (каскад)

Перечень `types` живёт в `Sport_log.md`. Менеджер видов — на вкладке «Трекер» (⚙ Виды спорта): добавить / переименовать / удалить. Каскад:
- **переименование** → меняется в `types`, во всех записях журнала и во всех файлах программ с этим видом (перезапись `Sport_log.md` + затронутых `Workouts/*.md`);
- **удаление** → убирается из `types`; у программ вид очищается (перезапись их файлов); записи журнала **сохраняют** историческую метку (диалог показывает, сколько записей/программ затронуто).

## Известные упрощения (v1, бэклог)
- Список читает все файлы программ при открытии вкладки (нет индекса) — при большом числе можно ввести индекс.
- Цвета привязаны к порядку видов в `types`: перестановка перечня сменит цвета. Палитра на 10 видов, дальше циклится.
- Сохранение — по явным действиям (добавил/удалил/сохранил), без дебаунса (живой ввод только в редакторе программы, пишется по кнопке).
