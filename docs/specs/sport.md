# spec: sport

> Формат — [guides/Sport.md](../guides/Sport.md). Решение — ADR [0006](../decisions/0006-sport-single-kind-two-tabs.md).

## Назначение
Геймификация спорта (направление Sport). Единый `kind`, две вкладки: **Трекер** и **Программы**.

## Данные
- Журнал `Sport_log.md`: frontmatter `types:` (перечень видов) + таблица `Date | Type | Program | Note`.
  `Program` — ссылка `[Название](Workouts/<имя>.md)`; при чтении берётся текст ссылки.
- Программы — редактируемые `.md` в `Workouts/`, у каждой `type` во frontmatter.

## Поведение (UI)
- **Трекер:** счётчики (всего / эта неделя / серия по неделям); «Отметить тренировку»
  (дата + тип-чипы + опц. программа + заметка; «＋ тип» добавляет вид); менеджер видов
  (⚙: добавить/переименовать/удалить); **календарь-хитмап** (26 недель, день делится на сегменты
  по числу тренировок, цвет = вид; палитра вне темы); **столбики по неделям** (12); журнал по дням (удаление).
- **Программы:** список (бейдж вида), создание/чтение(`marked`)/правка/удаление `.md`.

## Правила и крайние случаи
- Перечень `types` — общий. **Каскад:** переименование вида → меняется в `types`, журнале и файлах
  программ; удаление → убирается из `types`, у программ очищается, журнал сохраняет историческую метку.
- Серия — по неделям (учитывает отдых между тренировками).
- Удалённые виды (в истории) красятся серым.
- Переименование программы = write нового файла + delete старого.
