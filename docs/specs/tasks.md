# spec: tasks

> Формат — [guides/Tasks.md](../guides/Tasks.md). Решения — ADR [0005](../decisions/0005-tasks-file-per-task.md), [0008](../decisions/0008-tolerant-frontmatter.md).

## Назначение
Персональный таск-трекер (направление Tasks). Вкладки «Текущие» (`active/`) и «Архив» (`archived/`).

## Данные
Файл на задачу `Task-{N}.md`. Frontmatter: `title, type(simple|checklist|steps), status(todo|in_progress|done),
opened_at, closed_at, history[]`. Тело: описание (simple — тело; списочные — intro до списка) + список
`- [ ]`/`- [x]` (для steps — заметка `>` и вложения под шагом). Вложения — `Documents/Task-N/`.

## Поведение (UI)
- **Список (Текущие):** «Создать задачу» (модалка: заголовок + тип, форма зависит от типа),
  фильтры — сортировка по дате создания/смене статуса/id (+направление), фильтр по статусу.
  Карточка: id, тип, статус-бейдж, прогресс, даты.
- **Архив:** сортировка по `closed_at` (desc), пагинация (по 10).
- **Деталь:** правка заголовка/описания, переключатель статуса todo/in_progress, контент по типу
  (CRUD пунктов/шагов, перестановка ↑/↓ у steps, заметка и вложения шага), «Закрыть/Вернуть/Удалить»,
  раскрывашки «история статусов» и «.md».
- Сохранение: дебаунс ~700мс + по blur для текста; структурные действия — сразу.
- Вложения: бинарная загрузка в `Documents/Task-N/`, превью картинок, открытие по тапу.

## Правила и крайние случаи
- id = `max(active+archived)+1`. Закрытие = write archived + delete active (создаём папку перед PUT).
- Удаление задачи: файлы в `Documents/` остаются намеренно.
- Толерантность: неизвестные ключи frontmatter и свободный текст сохраняются.
- Описание есть у всех типов (у списочных = intro до списка).
