# spec: checklist

## Назначение
Список задач `- [ ]` / `- [x]` с прогресс-баром (только просмотр прогресса).

## Данные
`.md`-файл (берётся `<id>.md` в папке модуля или первый `.md`). Строки `- [ ]` / `- [x]`.

## Поведение (UI)
- Прогресс-бар (сегменты) + процент + счётчик `done / total`.
- Список пунктов с состоянием.

## Правила и крайние случаи
- Нет файла/пунктов → пустое состояние с подсказкой.
- В текущей версии отметка делается правкой файла (в Obsidian), апп показывает прогресс.

## Код
`js/kinds/checklist.js`: `renderChecklist`.
