# Growth — память проекта для Claude Code

Личный трекер саморазвития одного пользователя. PWA на ванильном JS (один файл `app/index.html`) + Markdown-данные на Mail.ru WebDAV через Cloudflare Worker (CORS-прокси). Хостинг — Cloudflare Pages с автодеплоем из этого репозитория.

## Сначала прочитай
- `@docs/Claude.md` — архитектура, инфраструктура, конвенции, все `kind` артефактов, модель синхронизации.
- `docs/config.reference.md` — снимок текущих направлений и артефактов (живой `config.md` — на Mail.ru).
- `docs/guides/<Направление>.md` — форматы артефактов конкретного направления (читай, когда работаешь над ним).

## Как здесь работать (контракт)
- **Код — один файл `app/index.html`** (ванильный JS, без сборки). Перед коммитом проверяй синтаксис: вытащи инлайновый `<script>` и прогони `node --check`.
- **Деплой приложения автоматический:** `git push` в `main` → Cloudflare Pages публикует `app/` на `growth-ao5.pages.dev`. Ручной деплой не нужен.
- **Воркер** — отдельно и редко: `cd worker && npx wrangler deploy`.
- **Данные живут на Mail.ru WebDAV, НЕ в репозитории** (живой `config.md`, словарь, роадмап, конспекты). Канал поставки данных — скрипт **`./sync-data.sh`** (НЕ git):
  - `./sync-data.sh pull` — скачать актуальные данные с Mail.ru в `data/` (зеркало, в .gitignore);
  - правишь файл в `data/` (например `data/GrowthApp/config.md`);
  - `./sync-data.sh push "<путь>"` — залить обратно на Mail.ru (или `push` без аргумента — всё из `sync-manifest.txt`).
  - Креды — в `.env` (из `.env.example`, в .gitignore). Если `.env` нет — попроси пользователя создать его с паролем приложения Mail.ru.
  - После изменения `config.md` обнови и снимок `docs/config.reference.md` (он в git, для ориентира).
- **Пароль Mail.ru НИКОГДА не коммитить** — он вводится в приложении и хранится на устройстве. Для локальных скриптов работы с данными — креды только из `.env` (он в `.gitignore`). Секрет воркера и логин уже публичны (зашиты в раздаваемый `index.html`).
- **Триггер документации:** при любом изменении состояния проекта (новый артефакт/направление, новый `kind`, смена схемы/конвенции) — обнови `docs/Claude.md`, нужный гайд в `docs/guides/` и снимок `docs/config.reference.md`. Эти файлы — источник истины для будущих сессий.

## Раскладка
`app/` — приложение (публикуется) · `worker/` — CORS-прокси · `docs/` — память проекта и гайды.
